# Security Policy

## Supported Versions
Only the **main branch** is actively supported for security updates.

## Reporting a Vulnerability
If you discover a security vulnerability, please report it responsibly:

- **Via GitHub Discussions:** You may open a discussion in the repository to report potential security concerns.
- **Via Email:** Send details privately to [business@kudagon.site](mailto:business@kudagon.site).
- **Via GitHub Security Advisories:** You can also use the GitHub Security Advisories feature to report issues.

## Disclosure Policy
- **Do not disclose vulnerabilities publicly** until we confirm and release a fix.
- We aim to **respond within 48 hours** of receiving a report.
- Once verified, we will work on a resolution and coordinate a safe disclosure timeline.

## Expectations
- Reporters should keep vulnerabilities **private** until a fix has been deployed.
- We will credit responsible reporters in release notes if desired.
